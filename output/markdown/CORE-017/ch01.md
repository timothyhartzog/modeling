# Chapter 1: Floating-Point Arithmetic and the Algebra of Rounding

## Motivation

On June 4, 1996, the maiden flight of the Ariane 5 rocket ended 37 seconds after launch when the vehicle veered off course and self-destructed. The root cause was a single floating-point conversion: a 64-bit velocity value was cast to a 16-bit signed integer, causing an overflow that crashed the inertial navigation system. The software had worked flawlessly on the Ariane 4, whose slower trajectory kept the value within 16-bit range. A modeling pipeline trusted blindly was destroyed by the gap between real arithmetic and machine arithmetic.

This chapter develops the mathematical framework for understanding that gap. Every numerical method in this curriculum — every matrix factorization, every ODE integrator, every optimization step — operates not on real numbers but on a finite, discrete approximation. Without understanding this approximation, a modeler cannot distinguish a genuine physical instability from a numerical artifact, cannot predict when a computation will lose all significant digits, and cannot choose between two algorithms that produce different answers for the same problem.

The central distinction of this chapter is between the **conditioning** of a mathematical problem (an intrinsic property, independent of any algorithm) and the **stability** of a computational algorithm (a property of a specific implementation). A well-conditioned problem solved by a stable algorithm produces reliable results; any other combination can fail silently, producing plausible-looking garbage.

> **Prerequisites.** Undergraduate calculus and linear algebra. Definition of matrix norms from CORE-002, Chapter 1. Familiarity with Julia basics from CORE-004, Chapter 1.

## The IEEE 754 Floating-Point System

### Representation

A normalized IEEE 754 double-precision floating-point number has the form

x = (-1)^s × (1.b₁b₂...b₅₂) × 2^(e - 1023)

where s ∈ {0, 1} is the sign bit, b₁...b₅₂ are the 52 bits of the significand (mantissa), and e is an 11-bit biased exponent with 1 ≤ e ≤ 2046. The leading "1." is implicit (the "hidden bit"), giving 53 effective bits of precision.

> **Definition 1.1 (Machine Epsilon).** The machine epsilon ε_mach is the smallest positive floating-point number such that fl(1 + ε_mach) > 1 in the floating-point system. For IEEE 754 double precision, ε_mach = 2^(-52) ≈ 2.22 × 10^(-16).

The unit roundoff u is defined as u = ε_mach / 2 = 2^(-53) ≈ 1.11 × 10^(-16) under round-to-nearest-even (the default rounding mode). The unit roundoff bounds the relative error of representing any real number in the float system.

> **Theorem 1.1 (Fundamental Rounding Theorem).** For any real number x in the normalized range of IEEE 754 double precision, the floating-point representation fl(x) satisfies
>
> fl(x) = x(1 + δ), where |δ| ≤ u.
>
> *Proof.* Let x > 0 lie in the interval [2^e, 2^(e+1)) for some integer e. The adjacent floating-point numbers in this interval are spaced 2^e × 2^(-52) = 2^(e-52) apart. Under round-to-nearest, the rounding error is at most half this spacing: |fl(x) - x| ≤ 2^(e-53). Thus |fl(x) - x| / |x| ≤ 2^(e-53) / 2^e = 2^(-53) = u. □

### Special Values and Subnormals

IEEE 754 reserves exponent fields e = 0 (subnormal numbers and ±0) and e = 2047 (±∞ and NaN). Subnormal numbers fill the gap between zero and the smallest normalized number, preventing a sudden jump from zero to 2^(-1022) ≈ 2.225 × 10^(-308). This "gradual underflow" ensures that x ≠ y if and only if fl(x) - fl(y) ≠ 0, a property critical for convergence tests in iterative algorithms.

> **⚠ Pitfalls and Misconceptions**
>
> 1. **"Double precision gives 16 digits of accuracy."** This is only true for the representation of a single number. After a chain of operations, accumulated rounding errors can consume most of those digits. A single subtraction of nearly equal numbers can destroy all significant digits in one step (see catastrophic cancellation below).
>
> 2. **"If my answer looks reasonable, floating-point errors are negligible."** Floating-point errors can produce results that look perfectly plausible but are completely wrong. The classic example: naive computation of the sample variance can return negative values for constant data due to cancellation in the sum-of-squares formula.

## Forward and Backward Error Analysis

> **Definition 1.2 (Forward Error).** For a computation that produces an approximate answer ŷ to a problem with true answer y, the absolute forward error is |ŷ - y| and the relative forward error is |ŷ - y| / |y|.

> **Definition 1.3 (Backward Error).** The backward error of an approximate answer ŷ is the smallest perturbation Δx to the input x such that ŷ is the exact answer to the perturbed problem f(x + Δx) = ŷ.

The backward error perspective, pioneered by Wilkinson, is the foundation of modern numerical linear algebra. Rather than asking "how far is my answer from the truth?" (forward error, which depends on the unknown truth), we ask "for what nearby problem is my answer exact?" (backward error, which we can compute). If the backward error is small (comparable to roundoff in the input data) and the problem is well-conditioned, then the forward error must also be small.

> **Theorem 1.2 (Forward-Backward Error Relationship).** If an algorithm produces a result ŷ = f(x + Δx) with backward error |Δx|/|x| ≤ η, and the problem has condition number κ, then the relative forward error satisfies
>
> |ŷ - y| / |y| ≤ κ · η + O(η²).
>
> *Proof.* By Taylor expansion: f(x + Δx) = f(x) + f'(x)Δx + O(Δx²). Thus ŷ - y = f'(x)Δx + O(Δx²), and |ŷ - y|/|y| = |f'(x)||Δx|/|f(x)| + O(Δx²) = κ · |Δx|/|x| + O(Δx²/x²). □

This theorem is the Rosetta Stone of numerical analysis: it factorizes error into conditioning (κ, intrinsic to the problem) times backward error (η, controlled by the algorithm).

## Conditioning of Linear Systems

> **Definition 1.4 (Condition Number of a Matrix).** For a nonsingular matrix A ∈ ℝ^(n×n) and a consistent matrix norm, the condition number is
>
> κ(A) = ‖A‖ · ‖A⁻¹‖.
>
> For the 2-norm, κ₂(A) = σ_max(A) / σ_min(A), the ratio of the largest to smallest singular values.

> **Theorem 1.3 (Perturbation Bound for Linear Systems).** Let Ax = b and (A + ΔA)(x + Δx) = b + Δb. If ‖ΔA‖/‖A‖ < 1/κ(A), then
>
> ‖Δx‖/‖x‖ ≤ κ(A)/(1 - κ(A)‖ΔA‖/‖A‖) · (‖ΔA‖/‖A‖ + ‖Δb‖/‖b‖).
>
> To first order: ‖Δx‖/‖x‖ ≲ κ(A) · (‖ΔA‖/‖A‖ + ‖Δb‖/‖b‖).
>
> *Proof.* From (A + ΔA)(x + Δx) = b + Δb, expanding and using Ax = b:
> A·Δx = Δb - ΔA·x - ΔA·Δx,
> so Δx = A⁻¹(Δb - ΔA·x - ΔA·Δx).
> Taking norms: ‖Δx‖ ≤ ‖A⁻¹‖(‖Δb‖ + ‖ΔA‖·‖x‖ + ‖ΔA‖·‖Δx‖).
> Rearranging: ‖Δx‖(1 - ‖A⁻¹‖·‖ΔA‖) ≤ ‖A⁻¹‖(‖Δb‖ + ‖ΔA‖·‖x‖).
> Dividing by ‖x‖ and using ‖A⁻¹‖·‖A‖ = κ(A) yields the result. □

**Interpretation.** If κ(A) = 10^k, then solving Ax = b in double precision (where perturbations are at the u ≈ 10^(-16) level) yields a solution accurate to roughly 16 - k digits. If κ(A) = 10^(15), only one digit is reliable. If κ(A) = 10^(17), the computation is meaningless.

**Example 1.2 (The Hilbert Matrix: A Poster Child of Ill-Conditioning).**

```julia
# Demonstration: The Hilbert matrix H_{ij} = 1/(i+j-1) is notoriously ill-conditioned.
# Observe that cond(H) grows super-exponentially with n, causing the computed
# solution to lose all accuracy for moderate n.

using LinearAlgebra, Printf

"""
    hilbert_demo(n::Int)

Solve Hx = b for the n×n Hilbert matrix with known solution x = ones(n).
Report condition number and relative error.
"""
function hilbert_demo(n::Int)
    H = [1.0 / (i + j - 1) for i in 1:n, j in 1:n]
    x_true = ones(n)
    b = H * x_true
    x_comp = H \ b
    
    rel_error = norm(x_comp - x_true) / norm(x_true)
    κ = cond(H)
    digits_lost = log10(κ)
    
    @printf("n = %2d | κ(H) = %10.2e | rel error = %10.2e | digits lost ≈ %.1f\n",
            n, κ, rel_error, digits_lost)
end

println("Hilbert Matrix Conditioning Experiment")
println("="^72)
for n in [3, 5, 7, 10, 12, 14]
    hilbert_demo(n)
end
```

## Catastrophic Cancellation

> **Definition 1.5 (Catastrophic Cancellation).** Catastrophic cancellation occurs when two nearly equal floating-point numbers are subtracted, causing the relative error of the result to be much larger than the relative errors of the operands. If a = fl(â) = â(1 + δ_a) and b = fl(b̂) = b̂(1 + δ_b), then
>
> fl(a - b) = (â - b̂) + (â·δ_a - b̂·δ_b),
>
> and the relative error of the difference is approximately |â·δ_a - b̂·δ_b| / |â - b̂|, which blows up when â ≈ b̂.

**Example 1.3 (The Quadratic Formula: A Cautionary Tale).**

The roots of ax² + bx + c = 0 are x = (-b ± √(b² - 4ac)) / (2a). When b² ≫ 4ac, one root involves subtracting two nearly equal numbers.

```julia
# Demonstration: Catastrophic cancellation in the quadratic formula.
# For b >> sqrt(4ac), the smaller root computed via the standard formula
# loses nearly all significant digits. The stabilized formula avoids this.

"""
    quadratic_naive(a::Float64, b::Float64, c::Float64)

Compute roots of ax² + bx + c = 0 using the standard textbook formula.
Suffers from catastrophic cancellation when b² ≫ 4ac.
"""
function quadratic_naive(a::Float64, b::Float64, c::Float64)
    disc = sqrt(b^2 - 4a*c)
    x1 = (-b + disc) / (2a)
    x2 = (-b - disc) / (2a)
    return x1, x2
end

"""
    quadratic_stable(a::Float64, b::Float64, c::Float64)

Compute roots using the stabilized formula that avoids cancellation.
First compute the root with larger magnitude, then use Vieta's formula
x1 * x2 = c/a for the other root.
"""
function quadratic_stable(a::Float64, b::Float64, c::Float64)
    disc = sqrt(b^2 - 4a*c)
    # Choose the sign that avoids cancellation
    if b >= 0
        x1 = (-b - disc) / (2a)
    else
        x1 = (-b + disc) / (2a)
    end
    x2 = c / (a * x1)  # Vieta's formula
    return x1, x2
end

# Test case: x² - 10^8 x + 1 = 0
# True roots: 10^8 - 10^(-8) and 10^(-8) (approximately)
a, b, c = 1.0, -1e8, 1.0
x1_naive, x2_naive = quadratic_naive(a, b, c)
x1_stable, x2_stable = quadratic_stable(a, b, c)

x_true_small = 1e-8  # approximately c/(-b)
println("Small root:")
println("  Naive:      $x2_naive  (rel error: $(abs(x2_naive - x_true_small)/x_true_small))")
println("  Stabilized: $x2_stable (rel error: $(abs(x2_stable - x_true_small)/x_true_small))")
```

**Example 1.4 (Sample Variance: Naive vs Welford's Algorithm).**

```julia
# Demonstration: Computing sample variance two ways.
# The naive formula Var = (Σx² - (Σx)²/n)/(n-1) involves subtracting
# two large, nearly equal numbers when the mean is large relative to the spread.
# Welford's one-pass algorithm avoids this entirely.

"""
    var_naive(x::Vector{Float64})

Compute sample variance via the textbook two-pass sum-of-squares formula.
Numerically unstable when mean(x)² ≫ var(x).
"""
function var_naive(x::Vector{Float64})
    n = length(x)
    sum_x = sum(x)
    sum_x2 = sum(xi^2 for xi in x)
    return (sum_x2 - sum_x^2 / n) / (n - 1)
end

"""
    var_welford(x::Vector{Float64})

Compute sample variance via Welford's numerically stable one-pass algorithm.
Maintains a running mean and sum of squared deviations from the mean.
"""
function var_welford(x::Vector{Float64})
    n = length(x)
    mean = 0.0
    M2 = 0.0
    for i in 1:n
        delta = x[i] - mean
        mean += delta / i
        delta2 = x[i] - mean
        M2 += delta * delta2
    end
    return M2 / (n - 1)
end

# Test: data with large mean, small variance
data = 1e9 .+ randn(10_000)
println("True variance ≈ 1.0")
println("  Naive:   $(var_naive(data))")
println("  Welford: $(var_welford(data))")
println("  Julia:   $(var(data))")  # Julia uses a stable algorithm
```

> **⚠ Pitfalls and Misconceptions**
>
> 1. **"I'll just use BigFloat to avoid all floating-point issues."** BigFloat (arbitrary-precision arithmetic) does not eliminate rounding — it only gives you more digits before rounding occurs. The condition number of the problem remains the same. An ill-conditioned problem in BigFloat still loses digits proportionally; it just has more to lose. Moreover, BigFloat is 100-1000× slower than Float64 and breaks BLAS/LAPACK acceleration, making it impractical for large-scale computation.
>
> 2. **"Catastrophic cancellation only matters for subtraction."** Cancellation can be hidden inside library functions. For example, `log(1 + x)` for small x involves computing `1 + x` (which rounds away the information in x) and then taking the logarithm. Julia provides `log1p(x)` to compute log(1 + x) accurately. Similarly, `exp(x) - 1` for small x should use `expm1(x)`.

## The Higham Axioms: A Model for Error Analysis

Rather than tracking individual rounding errors through every operation, Higham's model provides a compact framework. The fundamental axiom of floating-point arithmetic states:

For any arithmetic operation ⊕ ∈ {+, -, ×, /}, and floating-point numbers a, b:

fl(a ⊕ b) = (a ⊕ b)(1 + δ), where |δ| ≤ u.

This model abstracts away the details of IEEE 754 rounding and captures the essential behavior: each operation introduces a relative perturbation bounded by the unit roundoff.

> **Definition 1.6 (The γ_n Notation).** For n floating-point operations, define
>
> γ_n = nu / (1 - nu).
>
> For nu ≪ 1 (which holds for any reasonable computation in double precision), γ_n ≈ nu. This notation simplifies error bounds: a product of n factors (1 + δ_i) with |δ_i| ≤ u satisfies ∏(1 + δ_i) = 1 + θ_n where |θ_n| ≤ γ_n.

**Example 1.5 (Error Analysis of Inner Product).**

The inner product s = Σᵢ aᵢbᵢ computed left-to-right satisfies:

fl(s) = Σᵢ aᵢbᵢ(1 + θ_{n-i+1})

where |θ_k| ≤ γ_k. The backward error interpretation: fl(s) is the exact inner product of a with a perturbed vector b̃ where |b̃ᵢ - bᵢ| ≤ γ_n|bᵢ|. The forward error satisfies:

|fl(s) - s| ≤ γ_n Σᵢ |aᵢ||bᵢ|.

Note the absolute values: the bound depends on Σ|aᵢ||bᵢ|, not |Σaᵢbᵢ|. When there is heavy cancellation (terms of alternating sign), the error can be much larger than the true result.

## Computational Laboratory

```julia
# Computational Laboratory: Empirical Study of Conditioning and Stability
#
# This laboratory ties together the chapter's concepts by empirically
# measuring how condition number, algorithm choice, and problem structure
# interact to determine solution accuracy.

using LinearAlgebra, Random, Printf
Random.seed!(42)

"""
    condition_experiment(; n_sizes=[10, 50, 100, 200],
                          n_trials=50, cond_targets=[1e2, 1e6, 1e10, 1e14])

For each target condition number and matrix size, generate random matrices
with prescribed condition number, solve Ax = b (with known solution),
and measure the achieved relative error. Compare to the theoretical bound
κ(A) × u.

Returns a matrix of results for plotting.
"""
function condition_experiment(; n_sizes=[10, 50, 100, 200],
                                n_trials::Int=20,
                                cond_targets=[1e2, 1e6, 1e10, 1e14])
    results = []
    
    for n in n_sizes
        for κ_target in cond_targets
            errors = Float64[]
            
            for trial in 1:n_trials
                # Generate matrix with prescribed condition number
                U, _ = qr(randn(n, n))
                V, _ = qr(randn(n, n))
                σ = range(1.0, 1.0/κ_target, length=n)
                A = Matrix(U) * Diagonal(collect(σ)) * Matrix(V)'
                
                # Known solution
                x_true = randn(n)
                b = A * x_true
                
                # Solve
                x_comp = A \ b
                rel_err = norm(x_comp - x_true) / norm(x_true)
                push!(errors, rel_err)
            end
            
            median_err = sort(errors)[div(n_trials, 2)]
            theoretical_bound = κ_target * eps(Float64)
            
            push!(results, (n=n, κ_target=κ_target, median_error=median_err,
                           theoretical=theoretical_bound,
                           ratio=median_err/theoretical_bound))
        end
    end
    
    # Print results table
    println("\nConditioning vs Solution Accuracy")
    println("="^80)
    @printf("%-5s  %-12s  %-12s  %-12s  %-8s\n",
            "n", "κ(A)", "median |Δx/x|", "κ·u bound", "ratio")
    println("-"^80)
    for r in results
        @printf("%-5d  %-12.2e  %-12.2e  %-12.2e  %-8.2f\n",
                r.n, r.κ_target, r.median_error, r.theoretical, r.ratio)
    end
    
    return results
end

# Run the experiment
results = condition_experiment()

# [FIGURE: Log-log scatter plot of condition number vs relative error,
#  with the theoretical bound κ·u shown as a dashed line.
#  Points should cluster near or below the line, confirming the theory.]

println("\n\nCancellation Experiment: Inner Product Accuracy")
println("="^60)
# Compare inner product accuracy with and without cancellation
for n in [100, 1000, 10_000]
    # Case 1: all positive (no cancellation)
    a_pos = rand(n)
    b_pos = rand(n)
    s_true_pos = sum(BigFloat.(a_pos) .* BigFloat.(b_pos))
    s_comp_pos = dot(a_pos, b_pos)
    err_pos = abs(Float64(s_true_pos - s_comp_pos)) / abs(Float64(s_true_pos))
    
    # Case 2: alternating sign (heavy cancellation)
    a_alt = rand(n) .* [(-1)^i for i in 1:n]
    b_alt = rand(n)
    s_true_alt = sum(BigFloat.(a_alt) .* BigFloat.(b_alt))
    s_comp_alt = dot(a_alt, b_alt)
    err_alt = Float64(s_true_alt) != 0.0 ?
        abs(Float64(s_true_alt - s_comp_alt)) / abs(Float64(s_true_alt)) : 0.0
    
    @printf("n = %5d | no cancel: %.2e | cancel: %.2e | ratio: %.1f×\n",
            n, err_pos, err_alt, err_alt / max(err_pos, 1e-20))
end
```
